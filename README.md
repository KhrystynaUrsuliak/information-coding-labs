# Залікова робота

## Кодування та алгоритми передачі інформації

---

# Мета роботи

Продемонструвати комплексне розуміння:

* ентропії джерел інформації;
* моделей джерел;
* оптимального кодування;
* ефективності стискання;
* передачі інформації через канал з помилками;
* компромісу між ефективністю та надійністю передачі.

---

# Загальна постановка задачі

Необхідно реалізувати комплексну систему аналізу, кодування, стискання та передавання інформації для **бінарного джерела даних**.

На відміну від лабораторних робіт, де окремі етапи виконувалися незалежно, у заліковій роботі всі компоненти повинні бути інтегровані в єдиний процес.

---

# Джерело інформації

У якості джерела інформації необхідно використати **бінарний файл**.

Допускаються:

* executable files
* images
* archives
* PDF documents
* audio fragments
* video fragments
* інші бінарні дані

---

# Важливо

Аналіз виконується **на рівні байтів**.

Алфавіт джерела:

```text id="55eiv7"
{0, 1, 2, ..., 255}
```

Кожен байт розглядається як окремий символ джерела інформації.

---

# Загальні вимоги

Робота повинна:

* використовувати результати попередніх лабораторних;
* містити повний цикл обробки інформації;
* включати експериментальні дослідження;
* містити аналіз ефективності системи;
* містити таблиці та графіки.

---

# Основне завдання

Необхідно реалізувати систему:

```text id="ywsr4f"
бінарне джерело
→ аналіз ентропії
→ модель джерела
→ Huffman coding
→ стиснення
→ noisy channel
→ декодування
→ аналіз втрат
```

---

# Частина 1. Аналіз бінарного джерела

## Необхідно

Для вибраного файлу:

1. Обчислити:

$$
H_0
$$

— ентропію байтів.

---

2. Обчислити:

$$
H_1
$$

— ентропію пар байтів.

---

3. Обчислити:

* максимальну ентропію;
* надлишковість джерела.

---

4. Побудувати гістограму частот байтів.

---

## Вимоги

Розмір файлу:

```text id="1h1q2e"
не менше 1 МБ
```

---

# Частина 2. Моделювання джерела

## Необхідно

Побудувати:

* модель без пам’яті;
* модель з пам’яттю 1-го порядку.

---

## Для обох моделей

Необхідно:

* згенерувати байтові послідовності;
* порівняти статистичні характеристики;
* порівняти ентропію.

---

# Частина 3. Huffman coding

## Необхідно

1. Побудувати код Хаффмана для байтів.

2. Реалізувати:

   * кодування;
   * декодування.

3. Обчислити:

$$
L = \sum p_i l_i
$$

4. Порівняти:

$$
H \quad \text{та} \quad L
$$

---

# Частина 4. Дослідження стискання

## Необхідно

Виконати стискання:

* початкового файлу;
* Huffman-кодованого файлу;
* випадкових байтових даних.

---

## Використати щонайменше два алгоритми

Наприклад:

* gzip
* xz
* bzip2
* zip
* 7z

---

## Обчислити

$$
C = \frac{S_{compressed}}{S_{original}}
$$

---

# Частина 5. Передача через noisy channel

## Необхідно

Реалізувати двійковий симетричний канал.

---

## Провести передачу:

### 1. Без захисного кодування

### 2. З repetition code

### 3. З власним методом захисту

Наприклад:

* parity bit
* Hamming code
* CRC
* інший алгоритм

---

# Частина 6. Комплексний експеримент

Необхідно дослідити повний цикл:

```text id="mq1hvw"
binary source
→ Huffman coding
→ compression
→ noisy channel
→ decoding
→ reconstructed data
```

---

# Необхідно оцінити

* коефіцієнт стискання;
* ефективність кодування;
* кількість помилок;
* втрати інформації;
* вплив шуму на відновлення даних.

---

# Дослідження параметрів системи

Провести експерименти для:

```text id="w7v8jp"
p = 0.01
p = 0.05
p = 0.1
```

та різних типів файлів.

---

# Обов’язкові графіки

Необхідно побудувати:

1. гістограму частот байтів;
2. ентропія vs тип файлу;
3. коефіцієнт стискання vs алгоритм;
4. кількість помилок vs ймовірність помилки;
5. ефективність кодування vs тип джерела.

---

# Рекомендовані типи файлів

Бажано дослідити:

| Тип файлу | Очікування           |
| --------- | -------------------- |
| TXT       | добре стискається    |
| BMP       | добре стискається    |
| PNG/JPG   | майже не стискається |
| ZIP/7z    | майже не стискається |
| EXE       | середня ентропія     |

---

# Додаткові бали

Можна отримати бонусні бали за:

* adaptive Huffman
* arithmetic coding
* Hamming(7,4)
* CRC
* Reed-Solomon
* веб-інтерфейс симуляції
* візуалізацію дерева Хаффмана
* інтерактивний noisy channel simulator

Студент повинен продемонструвати:

# Питання для аналізу

1. Чому різні типи файлів мають різну ентропію?
2. Чому ZIP-файли майже не стискаються повторно?
3. Чому випадкові дані мають високу ентропію?
4. Чому код Хаффмана наближений до оптимального?
5. Як шум впливає на передавання даних?
6. Чому захисне кодування збільшує надлишковість?
7. Який компроміс існує між:

   * стисканням;
   * швидкістю;
   * надійністю передачі?
